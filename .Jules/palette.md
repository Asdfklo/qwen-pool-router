## 2025-06-25 - [A11y and UX Enhancements for Telemetry Dashboard]
**Learning:** Invisible modal backdrops styled with opacity transitions are still focusable by default if not set to 'visibility: hidden' or 'display: none'. This breaks standard keyboard tab orders by capturing keyboard focus on completely transparent elements. Additionally, dialog/drawer overlays must provide explicit, keyboard-accessible Close controls within their borders, and visual selectors (like duration buttons) require dedicated focus-visible indicator outlines to prevent keyboard users from navigating blindly.
**Action:** Always pair 'opacity: 0' transitions with 'visibility: hidden' to remove invisible backdrops from the keyboard tab order. Ensure every dialog/modal has a clear close button. Include ':focus-visible' CSS outlines and proper ARIA labels for stateful or context-specific toggle controls.
